Resumão Completo dos Capítulos

Material de apoio com explicações mais detalhadas para cada tópico do roadmap.

1

Conceitos básicos de redes

Antes de estudar protocolos, IP e segurança, é importante entender o que é uma rede e por que ela existe.

O que é uma rede?

Uma rede é um conjunto de dispositivos conectados entre si para trocar informações, compartilhar recursos e permitir comunicação. Esses dispositivos podem ser computadores, celulares, servidores, impressoras, câmeras e outros equipamentos.

Exemplo: computadores de uma escola conectados para acessar a internet e uma impressora compartilhada.

LAN, WAN, MAN e WLAN

LAN é uma rede local, como a rede de uma casa, escola ou empresa. WAN é uma rede de longa distância, como a internet. MAN cobre uma área metropolitana, como uma cidade. WLAN é uma rede local sem fio, normalmente usando Wi-Fi.

Exemplo: o Wi-Fi da sua casa é uma WLAN dentro de uma LAN.

Cliente, servidor e host

Cliente é o dispositivo ou sistema que solicita um serviço. Servidor é quem fornece esse serviço. Host é qualquer dispositivo conectado à rede que possui identificação, como endereço IP.

Exemplo: ao acessar um site, seu navegador é o cliente e o computador que hospeda o site é o servidor.

Internet, intranet e extranet

A internet é a rede mundial de computadores. A intranet é uma rede privada usada dentro de uma organização. A extranet é uma extensão controlada da intranet para usuários externos, como parceiros ou fornecedores.

Exemplo: um portal interno de funcionários é uma intranet.
2

Modelos de rede

Modelos de rede ajudam a entender como os dados saem de um dispositivo e chegam até outro.

Modelo OSI

O modelo OSI divide a comunicação em 7 camadas: física, enlace, rede, transporte, sessão, apresentação e aplicação. Ele é muito usado para estudo, diagnóstico e entendimento da comunicação em rede.

Exemplo: cabo de rede fica relacionado à camada física; IP fica relacionado à camada de rede.

Modelo TCP/IP

O modelo TCP/IP é mais usado na prática e está diretamente ligado ao funcionamento da internet. Ele costuma ser dividido em camadas como acesso à rede, internet, transporte e aplicação.

Exemplo: HTTP trabalha na aplicação; TCP trabalha no transporte; IP trabalha na camada de internet.

Camadas e suas funções

Cada camada tem uma responsabilidade específica. Algumas cuidam do meio físico, outras do endereçamento, outras da entrega dos dados e outras da aplicação que o usuário utiliza.

Exemplo: quando você acessa um site, várias camadas trabalham juntas para carregar a página.

Encapsulamento de dados

Encapsulamento é o processo de adicionar informações aos dados conforme eles passam pelas camadas. Essas informações ajudam no envio, roteamento, controle e entrega da mensagem.

Exemplo: os dados de uma página web recebem informações de TCP, IP e da placa de rede antes de serem enviados.
3

Endereçamento IP

O endereço IP identifica dispositivos na rede e permite que eles se comuniquem.

IPv4 e IPv6

IPv4 é o formato mais tradicional, composto por quatro blocos numéricos, como 192.168.0.10. IPv6 é uma versão mais nova, criada para suportar uma quantidade muito maior de endereços.

Exemplo IPv4: 192.168.1.5. Exemplo IPv6: 2001:0db8:85a3::8a2e:0370:7334.

IP público e IP privado

IP privado é usado dentro de redes locais. IP público é usado para identificação na internet. Normalmente, os dispositivos da sua casa usam IP privado, enquanto o roteador usa um IP público para sair para a internet.

Exemplo: 192.168.0.20 geralmente é um IP privado.

Máscara de sub-rede

A máscara de sub-rede ajuda a definir qual parte do IP representa a rede e qual parte representa o dispositivo. Ela é importante para organizar redes e separar grupos de dispositivos.

Exemplo: em 192.168.1.10 com máscara 255.255.255.0, a rede geralmente é 192.168.1.0.

Gateway e DNS

Gateway é a saída da rede local para outras redes, geralmente o roteador. DNS é o serviço que traduz nomes de domínio em endereços IP.

Exemplo: ao digitar google.com, o DNS descobre qual IP corresponde a esse nome.
4

Equipamentos de rede

Os equipamentos de rede são responsáveis por conectar dispositivos, distribuir sinal e controlar o tráfego.

Switch

O switch conecta vários dispositivos dentro de uma mesma rede local. Ele encaminha os dados para o dispositivo correto usando endereços físicos, conhecidos como endereços MAC.

Exemplo: em um laboratório de informática, vários PCs podem estar ligados a um switch.

Roteador

O roteador conecta redes diferentes. Em casa, ele geralmente conecta sua rede local à internet e também pode distribuir endereços IP para os dispositivos.

Exemplo: o roteador da sua casa permite que celular, notebook e TV acessem a internet.

Access Point

O Access Point permite a conexão sem fio de dispositivos à rede. Ele é usado para criar ou ampliar uma rede Wi-Fi.

Exemplo: uma escola pode usar vários Access Points para distribuir Wi-Fi em salas diferentes.

Firewall

O firewall filtra o tráfego de rede com base em regras. Ele pode permitir ou bloquear conexões, ajudando a proteger a rede contra acessos indevidos.

Exemplo: uma empresa pode bloquear acessos externos a portas não utilizadas.
5

Protocolos de rede

Protocolos são regras de comunicação. Eles definem como os dados serão enviados, recebidos e interpretados.

HTTP e HTTPS

HTTP é usado para comunicação entre navegador e sites. HTTPS é a versão segura, usando criptografia para proteger os dados transmitidos.

Exemplo: sites de bancos e lojas virtuais devem usar HTTPS.

TCP e UDP

TCP é confiável e garante a entrega correta dos dados, sendo usado em navegação web, e-mail e transferências importantes. UDP é mais rápido, mas não garante entrega, sendo comum em chamadas de vídeo, jogos online e transmissões ao vivo.

Exemplo: baixar um arquivo usa TCP; uma chamada de voz pode usar UDP.

DNS

DNS traduz nomes de domínio para endereços IP. Sem DNS, seria necessário decorar números de IP para acessar sites.

Exemplo: o DNS transforma exemplo.com em um endereço IP do servidor.

DHCP

DHCP distribui automaticamente configurações de rede, como IP, máscara, gateway e DNS. Isso evita configurar cada dispositivo manualmente.

Exemplo: quando você conecta o celular ao Wi-Fi, ele recebe um IP automaticamente pelo DHCP.
6

Segurança em redes

Segurança em redes envolve proteger dados, controlar acessos e reduzir riscos de ataques.

Firewall e regras

O firewall funciona como uma barreira de controle. Suas regras definem quais conexões podem entrar ou sair da rede, com base em IP, porta, protocolo ou aplicação.

Exemplo: liberar a porta 443 para HTTPS e bloquear portas desnecessárias.

VPN

VPN cria uma conexão segura entre o usuário e uma rede ou servidor. Ela é muito usada para acesso remoto seguro e para proteger dados em redes não confiáveis.

Exemplo: um funcionário acessando a rede da empresa de casa usando VPN.

Criptografia

Criptografia transforma dados em uma forma protegida, dificultando que terceiros entendam as informações mesmo que consigam interceptá-las.

Exemplo: HTTPS usa criptografia para proteger senhas e dados enviados em formulários.

Boas práticas básicas

Algumas boas práticas são: usar senhas fortes, atualizar sistemas, trocar senhas padrão de roteadores, segmentar redes, usar firewall, monitorar acessos e evitar serviços expostos sem necessidade.

Exemplo: trocar a senha padrão do roteador evita acessos não autorizados.
